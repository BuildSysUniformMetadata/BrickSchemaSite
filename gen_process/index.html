Development

Brick is created from collective efforts. Brick schema is generated through following steps.

Processes

  1. Identify necessary terminologies from existing building metadata and modern applications. (Building application list).
  2. Organize the terminologies into Tags/Tagsets in a spreadsheet. (Schema Engineering Sheet)
  3. A script converts the sheet to Turtle files. (Brick schema generator)
  4. Generated schema will be shared via Getting Brick

Tools

The Brick examples and most of our development has used the Python rdflib library.

Issue Tracking

We welcome any comments through Github issue page (Issues). Issues will be reviewed regularly by a board meeting. In particular, we are pleased to listen to you for missing Tagsets and inaccurate definitions.
